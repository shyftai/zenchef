Context

Building on strong foundations

The scoring agent built in August 2025 was impressive work for its time. It processed a 15,993-restaurant TAM through Clay, applied ICP-based scoring, and delivered actionable prospect lists. That system did the job and delivered real results.

But six months of advancement in AI tooling, data enrichment methods, and infrastructure options means we can now do significantly more — with less operational overhead, better accuracy, and at a fraction of the cost.

Methodology

10-factor ICP scoring, verified by win/loss data

Every scoring weight is derived from actual Zenchef Netherlands win/loss analysis — not assumptions. The scoring model uses a 0-to-max approach for each dimension, then normalizes to a 0–100 scale.

Data Access

Talk to your data with Claude

Instead of filtering spreadsheets or writing SQL, the MCP server lets anyone on the team ask questions in plain language. Claude has full context of the scoring methodology and can explain why any restaurant scored the way it did.

Example conversation

You: "Show me the top 10 restaurants in Rotterdam that don't have a booking system and have an email address."

Claude: Uses outreach_candidates to return 10 restaurants sorted by V4 score, with email addresses, ratings, categories, and a brief explanation of each score.

You: "Why did Restaurant De Kas score higher than Blauw?"

Claude: Uses score_breakdown to compare both restaurants dimension by dimension, highlighting the specific factors where one outperforms the other.

Available tools

search_restaurants Multi-filter search by name, city, category, cuisine, score tier, booking system, contact availability
restaurant_profile Full detailed profile with all enriched data for a specific restaurant
score_breakdown Component-by-component V4 score explanation with reasoning per factor
dashboard High-level summary: tier distribution, top cities, booking systems, categories
outreach_candidates Sales-ready prospect lists with contact info, filterable by city and requirements
compare_clay_scores V4 vs Clay V2/V3 deviation analysis to validate scoring improvements
check_signals Detect milestones, growth patterns, competitive moves, and decay indicators

Salesforce integration

Lemlist integration

Hot prospects with email addresses automatically enter personalized outreach sequences. The scoring breakdown provides context for AI-generated messaging that references specific restaurant characteristics — category, location, current tech stack.

Signal-driven outreach

When the living database detects a signal — a restaurant crossing 500 reviews, switching from The Fork, or launching a new website — it automatically triggers a personalized outreach sequence with that specific trigger as the hook.

Continuous monitoring

Monthly re-scraping of all 15,993 TAM restaurants. The snapshot system tracks rating changes, review growth, booking system switches, and new restaurant discoveries. Every change is a potential sales trigger.

The end goal: An automatic enrichment, analysis, and campaign launcher in one. Without having to work and maintain Clay tables. The entire pipeline runs autonomously — new restaurants are discovered, profiled, scored, and entered into outreach sequences without human intervention.
Getting Started

MCP server setup

Connect the restaurant intelligence database to Claude Desktop in five steps. Once connected, anyone on the team can query the data through natural language.

Create a Supabase project

Go to supabase.com → New Project. The free tier handles 500+ restaurants easily.

Run the migration

Open the SQL Editor in Supabase and paste the contents of supabase_migration.sql. This creates the restaurants, snapshots, signals, and outreach tables.

Configure credentials

Copy .env.example to .env and fill in your Supabase URL and service role key from Settings → API.

Upload the data

Run node upload_to_supabase.js to load all 500 profiled restaurants into the database.

Connect to Claude Desktop

Add the MCP server to your Claude Desktop configuration:

// Claude Desktop → Settings → Developer → Edit Config { "mcpServers": { "zenchef-intelligence": { "command": "node", "args": ["C:/path/to/poc/mcp-server/index.js"], "env": { "SUPABASE_URL": "https://your-project.supabase.co", "SUPABASE_SERVICE_KEY": "your-service-role-key" } } } }

After restarting Claude Desktop, you'll see the Zenchef tools available in the tool picker. Start by asking "Show me the dashboard" to see an overview of all 500 restaurants.
